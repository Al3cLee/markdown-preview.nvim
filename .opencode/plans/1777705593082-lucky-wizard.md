## Feature 3: Sticky Lang Badge and Copy Button on Code Blocks

### Problem

`#content pre` has both `position: relative` and `overflow-x: auto`. The lang badge (`::before` pseudo-element) and copy button (`.copy-btn`, appended as a child of `<pre>`) use `position: absolute`, which anchors them to the `<pre>`'s *scrollable content area* — not its visible viewport. When the code block's content is wider than the window and the user scrolls horizontally, the badge and button scroll off with the content.

### Fix: wrapper element approach

The standard solution is to separate the two concerns:
- A **non-scrolling wrapper** (`<div class="code-wrap">`) holds `position: relative` and contains the badge and copy button as `position: absolute` children — these never scroll.
- The `<pre>` inside the wrapper handles only `overflow-x: auto`; it no longer has `position: relative`.

Because the lang badge is currently a `::before` pseudo-element on `<pre>` (CSS-generated content), it can't be independently positioned relative to a parent. The simplest fix is to replace the pseudo-element with a real `<span class="code-lang">` DOM element that lives inside `.code-wrap` (not inside `<pre>`). The `attachCopyButtons()` JS function already runs post-render and is the right place to inject both elements.

### Files to modify

`assets/index.html` only — CSS + JS, no Lua changes.

### JS changes (`attachCopyButtons`)

Current function appends `.copy-btn` into `<pre>`. Change it to:

1. Check if the `<pre>` has already been wrapped (guard: `pre.parentElement.classList.contains('code-wrap')`).
2. Create a `<div class="code-wrap">`, insert it before `<pre>` in the DOM, then `appendChild(pre)` into it.
3. If `pre.dataset.lang` is set, create `<span class="code-lang">` with `textContent = pre.dataset.lang` and `appendChild` it to the wrapper.
4. Create the copy button and `appendChild` it to the **wrapper** (not `<pre>`).

```js
function attachCopyButtons() {
    contentEl.querySelectorAll('pre.hljs').forEach(pre => {
        // Already processed
        if (pre.parentElement && pre.parentElement.classList.contains('code-wrap')) return;

        // Create wrapper
        const wrap = document.createElement('div');
        wrap.className = 'code-wrap';
        pre.parentNode.insertBefore(wrap, pre);
        wrap.appendChild(pre);

        // Lang badge (real DOM element, not ::before pseudo)
        if (pre.dataset.lang) {
            const lang = document.createElement('span');
            lang.className = 'code-lang';
            lang.textContent = pre.dataset.lang;
            wrap.appendChild(lang);
        }

        // Copy button
        const btn = document.createElement('button');
        btn.className = 'copy-btn';
        btn.title = 'Copy code';
        btn.innerHTML = COPY_ICON;
        btn.addEventListener('click', async (e) => {
            e.stopPropagation();
            const code = pre.querySelector('code');
            if (!code) return;
            try {
                await navigator.clipboard.writeText(code.textContent);
                btn.innerHTML = CHECK_ICON;
                btn.classList.add('copied');
                setTimeout(() => {
                    btn.innerHTML = COPY_ICON;
                    btn.classList.remove('copied');
                }, 1500);
            } catch (_) { }
        });
        wrap.appendChild(btn);
    });
}
```

# Plan: YAML Front-Matter Panel + Default Theme Config

## Summary

Two independent, frontend-focused features:
1. **YAML front-matter collapsible panel** — render front-matter as a hidden `<details>` block (a styled YAML code block) at the top of the preview, with a `hide_yaml` Lua config option to suppress it entirely.
2. **Default theme config** — expose a `default_theme` Lua config option (`"dark"` | `"light"`) that controls what theme the browser loads with.

No server logic changes. No new CDN dependencies (YAML is rendered as-is in a `<pre><code>` block, no parsing needed).

---

## Files to Modify

- `lua/markdown_preview/init.lua` — add two config keys + two template substitutions in `write_index`
- `assets/index.html` — add `<details>` panel HTML, CSS, JS wiring, and theme placeholder


To:
```js

if (window.markdownItFrontMatter) {
    md.use(window.markdownItFrontMatter, function(fm) { lastFrontMatter = fm; });
```

#### 6. JS — update front-matter panel after each render

In the `sync()` function, after `showContent()` is called (and before the morphdom block), add:

```js
// Update front-matter panel
const fmPanel = document.getElementById('fmPanel');
const fmContent = document.getElementById('fmContent');
if (fmPanel && fmContent) {
    if (!HIDE_YAML && lastFrontMatter && lastFrontMatter.trim()) {
        const wasOpen = fmPanel.open; // preserve open/closed state
        fmContent.querySelector('code').textContent = lastFrontMatter.trim();
        fmPanel.style.display = '';
        if (wasOpen) fmPanel.open = true;
    } else {
        fmPanel.style.display = 'none';
    }
    lastFrontMatter = null; // reset for next render
}
```

Note: `lastFrontMatter` is reset after each render because `md.render()` triggers the front-matter callback synchronously, so by the time we reach this code it already has the value (or null if there's no front-matter in this render).

The panel's open/closed state is preserved across re-renders: read `fmPanel.open` before updating, and restore it after if the panel was already visible.
